# Site Build Checklist

Before deploying:
1. Client name, offer, audience, and primary CTA are represented on the site.
2. Site type is chosen and justified: institutional simple, landing page, catalog, or custom.
3. Brand assets from `client-data/brand/` are used or explicitly marked unusable.
4. Pages listed in `client-data/client-form.json` exist or are listed as pending.
5. Images have appropriate alt text.
6. Mobile layout is checked at a narrow viewport.
7. Build command completes successfully.
8. No secrets are hardcoded into the repository.
9. Production deploy has explicit Lucas approval.

For substantial visual work:
1. Capture at least one desktop and one mobile screenshot when tooling is available.
2. Check that text does not overlap.
3. Check that primary actions are visible without scrolling on the homepage.
