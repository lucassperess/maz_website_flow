# Site Build Checklist

Before deploying:
1. Client name, offer, audience, and primary CTA are represented on the site.
2. Brand assets from `client-data/brand/` are used or explicitly marked unusable.
3. Pages listed in `client-data/client-form.json` exist or are listed as pending.
4. Images have appropriate alt text.
5. Mobile layout is checked at a narrow viewport.
6. Build command completes successfully.
7. No secrets are hardcoded into the repository.

For substantial visual work:
1. Capture at least one desktop and one mobile screenshot when tooling is available.
2. Check that text does not overlap.
3. Check that primary actions are visible without scrolling on the homepage.
